PROTOCOLS

HTTP methods

GET: The GET method is used to retrieve a resource from the server. When sending a GET request, the server returns the requested data in the response. GET requests should not have any side effects and are considered safe, meaning they should not change the server's state.

POST: The POST method is used to send data to the server for processing. It is often used to create new resources or submit data to perform a specific operation. POST requests can have side effects and change the server's state.

PUT: The PUT method is used to completely update an existing resource on the server. It replaces the existing data with the new data provided in the request.

DELETE: The DELETE method is used to delete a specified resource on the server. When sending a DELETE request, the server removes the specified resource.

PATCH: The PATCH method is used to partially update an existing resource on the server. It is typically used when you want to make a small modification to an existing resource without replacing the entire resource.

HTTP status code

1xx (Informational):
These status codes indicate that the server has received the request and is continuing the process. They are informational and not typically used in general web development.
Example: 100 Continue.

2xx (Success):
These status codes indicate that the request was successfully received, understood, and processed by the server.
Example: 200 OK: The request was successful.

3xx (Redirection):
These status codes indicate that further action is needed to complete the request.
Example: 304 Not Modified: The client can use its cached version of the resource.

4xx (Client Error):
These status codes indicate that there was an error on the client's side of the request.
Example: 404 Not Found: The requested resource could not be found on the server.

5xx (Server Error):
These status codes indicate that there was an error on the server's side of the request.
Example: 500 Internal Server Error: An unexpected error occurred on the server.

DNS list by types

A (Address) Record:
Associates a domain name with an IPv4 address. Used to map a hostname to an IP address.

AAAA (IPv6 Address) Record:
Associates a domain name with an IPv6 address. Used to map a hostname to an IPv6 address.

CNAME (Canonical Name) Record:
Creates an alias or alternate name for an existing record. Used to map multiple domain names to a single canonical (primary) domain name.

MX (Mail Exchanger) Record:
Specifies the mail server responsible for accepting incoming emails for a domain. Used to route email messages to the appropriate mail servers.

TXT (Text) Record:
Stores text-based information associated with a domain. Used for various purposes like domain verification, SPF (Sender Policy Framework) records, and more.

NS (Name Server) Record:
Specifies the authoritative DNS servers for a domain. Used to delegate the responsibility of resolving the domain to specific name servers.

SOA (Start of Authority) Record:
Provides administrative information about a DNS zone. Contains details such as the primary name server, contact email, serial number, and more.

PTR (Pointer) Record:
Performs the reverse mapping of an IP address to a domain name. Used in reverse DNS lookups.

SRV (Service) Record:
Specifies the location of specific services available in a domain. Used to define the hostname and port number of services like SIP, XMPP, and more.

CAA (Certification Authority Authorization) Record:
Specifies which certificate authorities (CAs) are authorized to issue SSL/TLS certificates for a domain. Used to enforce certificate issuance policies.

Identifying IP

To determine the IP address for domain names, you can use the "ping" command on the operating system's command line.

1. Open Terminal

2. At the command prompt, type "ping" followed by the domain name for which you want to determine the IP address.

As a result, you will receive a string containing the name and IP. For example for google.com:
PING google.com (216.58.209.206) 56(84) bytes of data.
